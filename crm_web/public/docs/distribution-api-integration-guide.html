SEEDCRM DISTRIBUTION API

分销接口联调说明

本文面向外部分销 / 电商系统研发、SeedCRM 集成管理员和联调人员，说明方案 B 的入站事件、签名、防重复、Outbox 回推、异常队列、回查对账和验收样本。

适用版本：方案 B V1 最后更新：2026-05-01 响应包：code / message / data

1. 接入概览

分销方案 B 的接口目标是：外部分销系统把已支付成交订单推送给 SeedCRM；SeedCRM 完成预约排档和门店履约后，再把 crm.order.used 异步回推给外部分销系统。
1. 外部支付 分销小程序或电商系统完成付款。
2. 入站事件 调用 POST /open/distribution/events。
3. 入站服务 统一经 DistributionEventIngestService。
4. 本地主链 创建或匹配 Customer，创建 Order(paid)。
5. 门店履约 预约排档，生成并完成 PlanOrder。
6. 回推完成 Outbox 异步回推 crm.order.used。 
ExternalDistributionPaidOrder -> Customer -> Order(paid) -> PlanOrder -> scheduler_outbox_event -> crm.order.used
分销已支付订单不进入 Clue；SeedCRM 不处理外部分销佣金、提现、退款资金流；Open API / Scheduler 都不得绕过入站服务直接写 Customer / Order / PlanOrder。

2. 环境与权限

项目 说明
后台接口前缀 本地默认 http://127.0.0.1:8080；前端代理默认 /api。
Open API 入口 外部分销系统调用 /open/distribution/events，不需要后台登录 token，但需要渠道配置和签名规则。
后台调试接口 后台用户登录后使用 X-Auth-Token 调用 /scheduler/* 接口。
配置入口 系统设置 -> 调度中心 -> 分销接口 -> 接入配置。
联调角色 INTEGRATION_ADMIN 配置和执行全局处理；INTEGRATION_OPERATOR 查看日志、异常和脱敏结果。

2.1 MOCK 与 LIVE

模式 用途 安全规则
MOCK 本地联调、回归测试、验收样本生成。 入站仍需要 X-Idempotency-Key 和事件字段；签名可跳过。
LIVE 真实外部分销系统联调或生产接入。 必须配置 AppId、AppSecret、回推地址、状态回查路径、对账路径，并携带签名头。

2.2 上线域名与 OpenAPI

配置项 用途 上线要求
SEEDCRM_SYSTEM_BASE_URL 系统后台访问域名，用于页面跳转、扫码服务单等前端入口。 生产环境必须配置为可访问的 HTTPS 域名。
SEEDCRM_API_BASE_URL API 域名，用于分销入站、回调地址、Swagger/OpenAPI 示例地址。 生产环境必须配置为外部分销系统可访问的 HTTPS 地址。
SEEDCRM_OPENAPI_ENABLED 是否启用 Swagger UI 和 /v3/api-docs/*。 生产可启用，但必须通过后台 token 访问；未授权用户不能看到接口定义。
SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN 本地访问 Swagger 是否允许免 token。 生产环境建议设置为 false。
Swagger UI 用于查看接口定义；正式联调请优先使用“接口调试 / 联调工作台”进行 dry-run 验证。生产环境不要把 Swagger/OpenAPI 直接暴露给公网未授权访问。

3. 已支付订单入站

POST /open/distribution/events

该接口接收外部分销已支付订单、取消、退款待处理、已退款等事件。所有写入动作统一进入入站业务服务。

3.1 请求头

Header 是否必填 说明
X-Partner-Code 建议必填 渠道编码，默认 DISTRIBUTION。如请求头和报文 partnerCode 同时存在，必须一致。
X-Idempotency-Key 必填 幂等键，页面展示为“防重复编号”。禁止使用手机号作为订单幂等依据。
X-Timestamp LIVE 必填 签名时间，允许 10 分钟窗口。建议 ISO-8601 UTC 时间。
X-Nonce LIVE 必填 随机串，用于防重放。同渠道已验证 nonce 不能重复使用。
X-Signature LIVE 必填 HMAC-SHA256 签名，规则见“签名与防重放”。

3.2 支持事件

eventType SeedCRM 行为 备注
distribution.order.paid 创建或匹配 Customer，创建 Order(paid)。 唯一允许创建订单的事件。
distribution.order.cancelled 只能更新已有订单的外部状态或进入异常队列。 不能反向创建订单。
distribution.order.refund_pending 记录外部退款待处理状态，必要时进入异常队列。 外部退款资金流仍由外部分销系统处理。
distribution.order.refunded 更新已有订单外部退款状态，或按冲突进入异常队列。 不得绕过入站服务直接改核心表。
distribution.promoter.synced / distribution.member.updated V1 仅记录事件日志。 不创建核心模型，不触发分佣提现。

3.3 请求体字段

字段 必填 说明
eventType 是 事件类型，例如 distribution.order.paid。
eventId 是 外部事件 ID，用于日志追踪和重复事件识别。
partnerCode 建议 渠道编码，默认 DISTRIBUTION。
member.externalMemberId paid 必填 外部会员 ID，与 partnerCode 一起用于会员身份匹配。
member.phone paid 必填 会员手机号，用于 Customer 匹配。不能作为订单幂等键。
member.role 否 会员身份展示字段，例如 leader、member，仅用于展示 / 画像。
promoter.externalPromoterId 否 外部分销者 ID，仅记录外部快照，不参与 SeedCRM 分佣提现。
order.externalOrderId paid 必填 外部订单号，与 partnerCode 共同作为订单唯一依据。
order.externalTradeNo 建议 外部支付流水号，用于对账和冲突排查。
order.type 建议 deposit 或 coupon，V1 本地默认可映射为团购券 / 定金类订单。
order.amount paid 必填 已支付金额，单位为分。系统入库时转换为元并保留两位小数。
order.status 建议 外部状态，例如 paid、cancelled、refund_pending、refunded。
rawData 是 外部原始报文快照，用于追踪、补偿、对账和审计。

4. 签名与防重放

LIVE 模式下，入站事件必须携带时间戳、随机串和签名。

项目 规则
签名算法 HMAC-SHA256
签名密钥 分销接口配置中的 AppSecret / clientSecret
入站签名原文 X-Timestamp + "|" + X-Nonce + "|" + X-Idempotency-Key + "|" + rawRequestBody
时间窗口 允许 10 分钟偏差，超出会返回时间戳异常。
nonce 防重放 同渠道已验证 nonce 不能重复使用。 
// 伪代码
source = timestamp + "|" + nonce + "|" + idempotencyKey + "|" + rawJsonBody
signature = hmacSha256Hex(clientSecret, source)

headers = {
  "X-Partner-Code": "DISTRIBUTION",
  "X-Idempotency-Key": idempotencyKey,
  "X-Timestamp": timestamp,
  "X-Nonce": nonce,
  "X-Signature": signature
}
生成签名时必须使用原始请求体字符串。字段顺序、空格、换行变化都会改变签名结果，联调时建议先固定 JSON 序列化方式。

5. 防重复规则

场景 结果 说明
同 X-Idempotency-Key 或同 eventId，报文一致 DUPLICATE 系统返回成功并忽略重复事件，不重复创建订单。
同 X-Idempotency-Key 或同 eventId，报文不一致 EXCEPTION_QUEUED 进入异常队列，异常类型为重复报文冲突，需要人工核对内容指纹。
同 partnerCode + externalOrderId 已存在，关键字段一致 EXISTING 刷新外部状态快照，不重复创建订单。
同外部订单号已存在，但交易号、会员、类型或金额冲突 EXCEPTION_QUEUED 进入异常队列，显示冲突字段、本地值、外部值和处理建议。
上线前请检查“幂等健康”。若历史重复接口接收日志导致唯一索引暂未生效，需要先导出、人工确认、治理重复记录，再由管理员或 DBA 重建唯一索引。禁止直接删除 Customer / Order / PlanOrder 数据。

6. 履约回推

门店服务完成后，SeedCRM 先在本地完成 PlanOrder finished -> Order used，再写入 scheduler_outbox_event，由调度器异步调用外部分销系统。

6.1 回推事件

项目 值
事件类型 crm.order.used
回推地址 分销接口配置中的 callbackUrl，或 baseUrl + endpointPath
幂等键 providerCode:eventType:orderId:planOrderId
签名原文 timestamp + "|" + nonce + "|" + idempotencyKey + "|" + payload

6.2 回推 Header

X-Partner-Code: DISTRIBUTION
X-Idempotency-Key: DISTRIBUTION:crm.order.used:202:301
X-Trace-Id: outbox-188
X-Event-Type: crm.order.used
X-Timestamp: 2026-05-01T02:30:00Z
X-Nonce: 7f29c40a-91e7-4e58-b38b-bb2cb6a3401c
X-Signature: hmac_sha256_hex

6.3 回推 Body

{
  "eventType": "crm.order.used",
  "eventId": "crm.order.used_202_301",
  "occurredAt": "2026-05-01T10:30:00",
  "partnerCode": "DISTRIBUTION",
  "order": {
    "externalOrderId": "o_202605010001",
    "internalOrderId": 202,
    "orderStatus": "used",
    "localOrderStatus": "COMPLETED",
    "usedAt": "2026-05-01T10:29:58"
  },
  "planOrder": {
    "planOrderId": 301,
    "planOrderStatus": "finished",
    "finishedAt": "2026-05-01T10:29:58"
  }
}
Outbox 是异步机制。外部系统短时不可用不会回滚本地履约，失败会进入失败或死信状态，待修正配置或外部接口后重新入队。

7. 状态回查与对账

Scheduler 负责状态回查、对账拉取、失败重试和补偿同步。它只能读取外部状态或本地 MOCK 数据；若需要改变订单状态，必须生成 distribution.order.* 事件并调用 DistributionEventIngestService.replayFromScheduler。

任务 后台触发接口 说明
履约回推队列处理 POST /scheduler/outbox/process?limit=20 处理 PENDING / 到期 FAILED 回推事件。
异常重试队列处理 POST /scheduler/distribution/exceptions/process?limit=10 处理 RETRY_QUEUED 异常，可信重放入站服务。
状态回查 POST /scheduler/distribution/status-check/process?limit=20 读取外部订单状态，生成状态事件重放。
对账拉取 POST /scheduler/distribution/reconcile/process?limit=20 拉取外部对账记录，统一转成入站事件处理。

7.1 LIVE 外部接口要求

能力 配置字段 外部接口期望
状态回查 statusQueryPath SeedCRM POST 外部地址，请返回状态或标准事件。支持 paid/cancelled/refund_pending/refunded 映射。
对账拉取 reconciliationPullPath SeedCRM POST 外部地址，请返回 items / records / list / orders 或单条对象。
履约回推 callbackUrl 或 baseUrl + endpointPath 外部分销系统接收 crm.order.used，并按幂等键去重。

8. 调试与诊断

用途：预览字段映射、状态映射、Customer 匹配、Order 幂等结果，不落库。

入口：系统设置 -> 调度中心 -> 分销接口 -> 验收样本。

用于查看唯一索引状态、重复分组、建议保留记录、需核对记录和日志样例。

后台接口：GET /scheduler/monitor/summary?providerCode=DISTRIBUTION。

返回 Outbox、异常队列、幂等健康、最近调度批次和验收样本。

9. 错误码与排障

错误类型 常见原因 处理建议
PROVIDER_INVALID 分销渠道未配置、被停用或 partnerCode 不匹配。 检查分销接口配置、启用状态、请求头和报文中的渠道编码。
SIGNATURE_MISSING / SIGNATURE_INVALID LIVE 请求缺少签名头，或签名原文 / 密钥不一致。 核对 AppSecret、原始请求体、timestamp、nonce、idempotencyKey。
TIMESTAMP_INVALID 时间戳格式错误或超出 10 分钟窗口。 统一服务器时间，使用 ISO-8601 或秒 / 毫秒时间戳。
IDEMPOTENCY_MISSING 缺少 X-Idempotency-Key。 外部系统为每个业务事件生成稳定防重复编号。
DUPLICATE_PAYLOAD_CONFLICT 同幂等键或同事件 ID 发送了不同报文。 核对内容指纹，外部系统修正报文或生成新的幂等键后重推。
EXTERNAL_ORDER_CONFLICT 同外部订单号已存在，但交易号、会员、类型或金额冲突。 查看异常队列冲突字段，确认以本地值或外部值为准后再处理。
EXTERNAL_STATUS_CONFLICT 外部取消 / 退款事件与本地履约状态冲突。 由业务人员确认是否需要退款、冲正绩效或仅记录外部状态。

9.1 异常处理标准流程

查看异常 进入分销接口 -> 异常队列。
定位订单 复制外部订单号、本地订单号、追踪编号。
核对字段 查看冲突字段、本地值、外部值。
修复源头 修正外部数据、接口配置或状态映射。
重新入队 管理员触发重试，系统可信重放。
关闭记录 成功后自动已处理，或人工标记处理。

10. 上线验收

  • distribution.order.paid 可创建 / 匹配 Customer 并创建 Order(paid)。
  • 分销已支付订单不进入 Clue，不参与公海和客资分配。
  • 重复同报文返回成功且不重复创建订单。
  • 同幂等键不同报文进入异常队列。
  • 同外部订单号关键字段冲突进入异常队列并显示冲突字段。
  • 门店完成 PlanOrder 后生成 crm.order.used Outbox。
  • MOCK 模式 Outbox 可模拟成功，LIVE 模式按签名调用外部回调地址。
  • 失败回推可重新入队，超过重试进入死信。
  • 状态回查 / 对账拉取只能生成事件重放，不直接更新核心表。
  • 异常 OPEN / RETRY_QUEUED / HANDLED 状态可闭环。
  • 幂等健康能显示唯一索引风险和重复数据治理建议。
  • INTEGRATION_OPERATOR 查看敏感字段为脱敏结果。
  • PARTNER_APP 只能访问自身 partner_code 范围，不能触发全局批处理。
  • 验收样本页 5 类样本可展示：成功、失败、死信、异常、对账失败。
  • 系统设置 -> 基础配置 -> 域名配置保存后，分销接口页展示的入站、回查、对账和 Swagger 地址同步更新。
  • Swagger UI / OpenAPI JSON 仅允许管理员、集成管理员或集成操作员访问；普通业务角色和外部分销应用不能访问。
  • 生产环境变量已配置：SEEDCRM_SYSTEM_BASE_URL、SEEDCRM_API_BASE_URL、SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN=false。
若幂等唯一索引因历史重复接收日志暂未生效，上线前必须完成数据治理或形成书面风险确认。治理原则是导出、人工确认、备份 / 归档、重建索引、复查，禁止直接删除核心业务数据。禁止直接删除 Customer / Order / PlanOrder 数据。

附录：已废弃客资输出接口

GET/POST /open/distribution/leads

该接口保留为兼容兜底，当前方案 B 不再向外部分销系统输出 Clue 客资。调用后会返回空列表和废弃提示，不会创建 Clue / Customer / Order。
