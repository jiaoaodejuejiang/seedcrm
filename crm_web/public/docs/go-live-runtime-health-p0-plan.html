SeedCRM P0-7 系统上线联调与运行健康中心详细落地方案

建议产品侧命名为“上线准备中心”。它是实施、运维、集成管理员使用的上线任务清单、联调入口和运行健康门禁，不是新的业务操作中心。

P0-7 上线准备 OpenAPI / Swagger Mock / LIVE 健康检查 生产硬门禁

1. 目标与边界

  • 标准主链路仍然是 Clue -> Customer -> Order -> PlanOrder。
  • 分销 B 方案仍然是 ExternalDistributionPaidOrder -> Customer -> Order(paid) -> PlanOrder。
  • 上线准备中心只治理域名、环境、OpenAPI、Provider、Scheduler、配置版本、初始化、测试数据清理、运行健康和部署文档。
  • 上线准备中心不直接创建、修改、删除真实 Customer、Order、PlanOrder。
  • 真实第三方测试数据不能通过后台表单或 SQL 直接入核心表，必须走 Open API / Scheduler 统一入站服务。
  • 分销 paid order 联调不进入 Clue，只能预览或通过受控入站事务创建 / 匹配 Customer + Order(paid)。
  • 状态回查、对账拉取、异常重放不得直接更新核心业务表，必须生成 distribution.order.* 事件并复用入站服务。
  • 生产环境禁止真实清理业务数据，这是硬门禁，不能被配置开关或确认词绕过。
  • Swagger/OpenAPI 在生产环境不能匿名裸开，必须受开关、角色、Token、IP 白名单或 VPN 约束。

6. 流程

6.1 上线准备步骤

  1. 部署检查：检查 backend、frontend、MySQL、Docker Compose、端口、镜像版本、最近部署时间。
  2. 域名配置：检查系统基础域名、API 域名、HTTPS、localhost、127.0.0.1、外部可达性。
  3. 环境确认：检查 TEST / STAGING / PROD、MOCK / LIVE、OpenAPI 开关、清理测试数据开关。
  4. 接口联调：展示入站地址、Swagger/OpenAPI、签名样例、字段映射、状态映射和 dry-run。
  5. MOCK 验证：验证 paid、重复 paid、cancelled、refund_pending、refunded、Outbox 模拟、异常队列闭环。
  6. LIVE 检查：检查 AppId、Secret、token、auth_code 时效、timestamp、nonce、signature、IP 白名单、状态回查、对账路径。
  7. 健康检查：检查登录、菜单权限、调度任务、队列积压、死信、幂等健康、敏感字段脱敏。
  8. 上线结论：输出可上线、需确认、不可上线，并生成检查报告。

6.2 Provider 联调流程

  1. 从 API 域名拼接回调地址和 OpenAPI 地址。
  2. 选择 Provider 和接口模板。
  3. 执行 dry-run，验证验签、timestamp、nonce、防重放、字段映射、状态映射、幂等。
  4. 分销 paid order dry-run 只能预览 Customer 匹配和 Order(paid) 幂等，不进入 Clue。
  5. refund / cancel dry-run 本地无订单时不得反向创建 Order。
  6. dry-run 通过后进入 MOCK 验证。
  7. 切 LIVE 前执行 prepare，展示影响范围和阻断项。
  8. confirm 后写配置版本、审计和运行健康检查记录。

6.3 Scheduler 联调流程

  1. 执行任务 dry-run，确认不写核心业务表、不调用真实外部接口。
  2. 状态回查、对账拉取只生成 distribution.order.* 预览事件。
  3. 异常重放 dry-run 只读取异常记录原 payload，不允许用户提交新 payload。
  4. Outbox dry-run 只预览 crm.order.used 回推 payload、签名和目标地址。
  5. 真实执行必须跳转调度与队列中心，走 prepare / confirm 和调度锁。

6.4 初始化与清理测试数据流程

  1. 初始化前展示当前环境、目标环境、域名、是否切回 MOCK、是否停用调度。
  2. 生产初始化需要 INIT_PROD_SYSTEM 确认词，且不会自动切回 MOCK。
  3. 测试数据初始化必须记录批次、环境、场景、来源和关联对象。
  4. 清理前必须执行 dry-run，展示将清理的表、数量、批次和影响范围。
  5. 真实清理需要确认词、后端开关、非 PROD 环境、最近一次 dry-run 结果一致。
  6. PROD 环境真实清理永远拒绝。
  7. 清理后生成报告和审计，不能静默删除。

6.5 上线后健康观测流程

  1. 定时采集 backend、frontend、MySQL、OpenAPI、Scheduler、Provider、Outbox、异常队列健康快照。
  2. 展示回调成功率、任务成功率、Outbox 积压、dead-letter、异常队列、配置版本状态。
  3. 出现阻断项时推送到 P0-6 工作台，提醒 admin / integration_admin / integration_operator。
  4. 配置异常通过配置版本 rollback 恢复，不直接手改数据库。

7. 页面体验设计

首屏结论

  • 顶部环境条常驻：当前环境、系统域名、API 域名、OpenAPI 状态、清理保护、最近检查时间。
  • 上线结论区分可上线、需确认、不可上线。
  • 不可上线时禁用切 LIVE、真实联调、真实清理等高风险动作。

步骤式导航

  • 部署检查、域名配置、环境确认、接口联调、MOCK 验证、LIVE 检查、健康检查、上线结论。
  • 每一步只做一类事情，失败项展示原因、影响和建议动作。
  • 右侧固定显示下一步、阻断项、跳转入口、最近操作记录。

高风险确认

  • 切 LIVE、开启 OpenAPI、生产初始化、清理测试数据、真实联调、配置发布、回滚必须二次确认。
  • 确认弹窗展示环境、差异预览、影响范围、确认原因和确认词。
  • OpenAPI 临时开启应填写关闭时间或到期提醒。

文档与报告

  • 部署文档、接口文档、回调文档、联调指南、故障处理手册按环境聚合展示。
  • 检查报告可导出，包含检查时间、操作者、环境、通过项、警告项、阻断项。
  • 技术字段默认折叠，敏感值脱敏。

10. 验收标准与禁止行为

验收标准

  • 能集中查看基础域名、API 域名、回调地址、OpenAPI/Swagger、Provider、Scheduler、Outbox、权限、配置版本状态。
  • 能一键执行上线检查，并输出通过、失败、风险和修复建议。
  • 能通过 OpenAPI 模板发起接口调试，自动生成签名示例和 traceId。
  • Mock / LIVE 状态全页面醒目可见，切 LIVE 必须二次确认。
  • auth_code 时效、timestamp、nonce、防重放、签名校验可被检查和 dry-run 验证。
  • 抖音联调只进入 Clue，不创建 Customer、Order、PlanOrder。
  • 分销 paid order 联调不进入 Clue，能预览 Customer 匹配和 Order(paid) 幂等。
  • 分销 refund / cancel 联调不得反向创建 Order。
  • Scheduler 测试默认 dry-run，真实执行必须二次确认。
  • 状态回查、对账拉取、异常重放必须复用统一入站服务。
  • 能一键初始化标准测试数据，并标记批次、环境、来源。
  • 能按批次清理测试数据，生产环境默认且永久禁止真实清理。
  • 所有联调、初始化、清理、调度测试、配置检查均有审计日志。
  • 敏感字段默认脱敏，密钥不明文展示。

禁止行为

  • 禁止新增核心业务模型或改变 Clue -> Customer -> Order -> PlanOrder。
