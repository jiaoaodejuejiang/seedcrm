SeedCRM 受控低代码 UI 呈现附录

本附录只做静态高保真示意，不新增后端 API，不新增前端业务页面。目标是让配置化平台未来页面长相、交互密度、风险提示和主链路边界在方案阶段就能被直观看到。

UI 附录 高保真静态示意 Element Plus 后台风格 不截图 不依赖启动系统

1. UI 总原则

  • 页面沿用当前 SeedCRM 的顶部一级导航、左侧二级菜单、白底卡片、状态 Tag、表格、抽屉、Tab、发布中心风格。
  • P0 阶段优先使用模板化配置、表单化属性和实时预览，不做复杂拖拽式万能低代码。
  • 配置平台只决定看什么、怎么展示、入口在哪里、是否允许操作，不直接写核心业务表。
  • 所有示意图都必须显示硬边界：不改变 Clue -> Customer -> Order -> PlanOrder，分销 paid order 不进入 Clue。
  • 高风险操作统一进入发布中心，不允许在设计器里直接生效。

3. 接口定义

本次只做文档，以下接口为后续落地的受控低代码接口引用。

接口页面用途约束
GET /lowcode/assets配置工作台、模板库、设计器资产列表。只读，敏感配置脱敏。
POST /lowcode/assets/save-draft保存菜单、列表、表单、按钮草稿。不影响线上运行。
POST /lowcode/validate发布前体检。校验主链路、权限、字段引用、脱敏和安全门禁。
POST /lowcode/preview-permission按角色/用户预览权限。必须叠加 RBAC + ABAC。
POST /lowcode/publish发布配置版本。写审计、失效缓存，可回滚。
GET /runtime-config/page/{pageCode}业务页面读取运行时 UI 元数据。只返回元数据，不执行业务写入。

4. 权限规则

配置平台 RBAC

  • ADMIN：全量配置、发布、回滚、审计。
  • INTEGRATION_ADMIN：Provider、Scheduler、接口联调配置。
  • INTEGRATION_OPERATOR：只读、dry-run、日志，不可发布。
  • STORE_MANAGER：门店模板、门店绑定、门店人员视图类配置。

运行时 ABAC

  • 数据范围固定为 self / team / store / all / partner。
  • 字段、按钮、列表和导出都必须叠加数据范围。
  • 手机号、raw_data、token、secret、auth_code、支付流水默认脱敏。
  • 前端参数不能提升数据范围。

5. 流程

  1. 配置人员从配置工作台进入配置向导。
  2. 选择目标、业务对象和模板，生成草稿。
  3. 进入设计器，通过表单化属性和实时预览调整页面。
  4. 保存草稿后执行校验、权限预览、影响预览和 dry-run。
  5. 高风险项进入发布中心，填写原因并二次确认。
  6. 发布后写入版本、审计并失效缓存。
  7. 异常时从发布记录生成回滚草稿，预览后发布为新版本。

6.1 配置化平台首页

首页不堆大段说明，第一屏只呈现环境、版本、草稿、风险和模板入口，方便管理员马上知道要处理什么。

6.2 配置向导页

向导从“业务目标”开始，不让实施人员直接面对 JSON。每一步都提示是否触碰主链路边界。

6.3 页面 / 列表设计器

设计器保持三栏布局：左侧字段和组件库，中间实时业务页面预览，右侧属性面板，底部固定草稿、校验、权限预览和提交发布。

6.4 菜单与权限配置页

菜单新增不再手填顶级菜单，使用树选择和权限码选择；发布前按角色/用户预览最终菜单、按钮、字段和数据范围。

6.5 服务单设计页

服务单设计延续当前模板库、门店绑定、效果预览思路。P0 重点是模板化、字段属性和预览，不做复杂海报级拖拽。

6.6 发布中心

发布中心承担高风险确认、差异对比、影响范围、版本发布和回滚。高风险不会在设计器直接上线。

7. 测试与验收

  • 文档打开后能直观看到 6 个未来页面的样式和操作重点。
  • 页面风格与当前后台一致，不像另一套系统。
  • UI 不是纯线框，有颜色、布局、状态 Tag、表格、抽屉、预览区和发布风险区。
  • 每个 UI showcase 都有明确目的：首页、向导、设计器、权限、服务单设计、发布中心。
  • 文档包含至少 6 个高保真 SVG 区块。
  • 文档仍包含 数据结构 / 接口定义 / 权限规则 / 流程 / 禁止行为。

8. 禁止行为

  • 禁止把 UI 示意理解为已经实现的业务功能。
  • 禁止通过低代码配置改变 Clue -> Customer -> Order -> PlanOrder。
  • 禁止配置按钮直接写 Customer、Order、PlanOrder。
  • 禁止让分销 paid order 进入 Clue。
  • 禁止 Scheduler / Provider 绕过 dry-run、发布中心、Outbox、异常队列和统一入站服务。
  • 禁止在审计、日志、diff、影响报告中明文展示 token、secret、auth_code、raw_data。
