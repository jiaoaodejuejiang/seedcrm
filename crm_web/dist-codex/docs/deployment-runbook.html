SeedCRM 快速部署与上线联调手册

覆盖 Docker Compose 部署、线上域名、OpenAPI/Swagger、上线工具、Scheme B 分销联调和回滚检查。

Docker Compose 上线工具 dry-run 不入库

1. Docker 快速启动

复制根目录 .env.example 为 .env，修改数据库密码和线上域名后执行：

docker compose build
docker compose up -d
docker compose logs -f backend
docker compose logs -f frontend

2. 必填环境变量

变量说明
SEEDCRM_SYSTEM_BASE_URL后台页面域名，用于扫码服务单、页面跳转。
SEEDCRM_API_BASE_URLAPI 域名，用于分销入站、企微/抖音/支付回调、Swagger/OpenAPI。
SEEDCRM_OPENAPI_ENABLED正式环境建议 false，需要联调时短期开启。
SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN正式环境必须 false。
SEEDCRM_ALLOW_CLEAR_TEST_DATA默认 false。只在确认是测试库、完成备份且需要真实清理时临时改为 true。

3. 上线工具

  1. 管理员进入 系统设置 / 基础配置 / 上线工具。
  2. 先看上线预检，确认 API 域名、系统域名、调度任务、分销配置、抖音配置。
  3. 测试/预发初始化确认词：INIT_SYSTEM；正式环境确认词：INIT_PROD_SYSTEM。
  4. 清理测试数据先 dry-run，真实清理必须输入 CLEAR_TEST_DATA，并显式开启 SEEDCRM_ALLOW_CLEAR_TEST_DATA=true；PROD 环境后端会阻断。

4. Scheme B 联调顺序

  1. 配置分销 partner、签名密钥、状态映射、订单类型映射、状态回查路径、对账路径。
  2. 先执行 dry-run 入站映射，确认不会写 Customer / Order / PlanOrder。
  3. 用 MOCK 正式链路验证 paid 事件创建或匹配 Customer，并创建 Order(paid)，不得进入 Clue。
  4. 验证 duplicate paid 不重复建单，cancelled/refund_pending/refunded 不反向创建订单。
  5. PlanOrder 完成后写 crm.order.used Outbox，再由调度异步回推外部分销系统。

5. 不可上线条件

  • 正式环境仍使用 127.0.0.1、localhost 或 HTTP 回调域名。
  • Swagger/OpenAPI 与本地免 Token 在正式环境开启且未做访问限制。
  • 分销 dry-run 会写核心业务表或真实调用外部接口。
  • 异常队列可以绕过入站服务直接修改 Customer / Order / PlanOrder。
  • 抖音 auth_code 已过期但仍继续换 token 或执行真实拉取。
