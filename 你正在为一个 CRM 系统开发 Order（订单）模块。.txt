你正在为一个 CRM 系统开发 Order（订单）模块。

【技术栈】
- Spring Boot 3
- MyBatis Plus
- MySQL
- 包名：com.seedcrm.crm

-----------------------------------
【一、业务背景】

系统核心模型：

Clue（线索） → Customer（客户） → Order（订单）

说明：
- Clue：未成交前线索（来自抖音）
- Customer：已成交用户（私域）
- Order：业务核心（到店服务 + 支付）

-----------------------------------
【二、订单核心能力】

实现以下功能：

1. 创建订单（支持 Clue 或 Customer）
2. 支付定金
3. 预约到店
4. 到店
5. 服务中
6. 完成订单
7. 取消订单

-----------------------------------
【三、数据库表设计】

表名：order_info

字段：

- id (PK)
- order_no (唯一订单号)
- clue_id
- customer_id
- type（订单类型：1定金单 2团购单 3补款单）
- amount（总金额）
- deposit（定金）
- status（订单状态）
- appointment_time
- arrive_time
- complete_time
- remark
- create_time
- update_time

-----------------------------------
【四、订单状态机（必须实现）】

CREATED（已创建）
→ PAID_DEPOSIT（已付定金）
→ APPOINTMENT（已预约）
→ ARRIVED（已到店）
→ SERVING（服务中）
→ COMPLETED（完成）

异常状态：
- CANCELLED
- REFUNDED

要求：

1. 状态必须严格流转（禁止跳跃）
2. 每个接口必须校验当前状态
3. 状态错误直接抛异常

说明：订单系统通常需要清晰的状态流转来保证业务一致性 :contentReference[oaicite:0]{index=0}

【七、核心业务逻辑要求】

1. 创建订单：
   - 生成 order_no
   - 状态 = CREATED

2. 支付定金：
   - 只能从 CREATED → PAID_DEPOSIT

3. 预约：
   - 只能从 PAID_DEPOSIT → APPOINTMENT

4. 到店：
   - 只能从 APPOINTMENT → ARRIVED

5. 完成：
   - 只能从 SERVING → COMPLETED
