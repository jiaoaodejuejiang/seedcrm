SeedCRM 受控低代码兼容主链路方案

结论：SeedCRM 可以引入“低代码”能力，但必须是受控低代码，也就是配置治理平台 + 运行时解释层。它用于减少高频系统需求反复改代码，不用于任意建模、任意 SQL、任意状态机编排，也不替代现有核心业务服务。

受控低代码 配置治理 兼容唯一主链路 RBAC + ABAC 发布前预览 可回滚可审计

1. 结论与边界

如果按“万能低代码”理解，让业务人员配置数据表、SQL、状态流、外部接口编排，就会破坏当前项目规则。SeedCRM 更适合做“受控低代码”：只开放菜单、权限、页面、表单、列表、按钮、字典、待办、Provider、Scheduler 等系统能力的配置，不开放核心业务模型和核心写入链路。

  • 标准客资链路固定为 Clue -> Customer -> Order -> PlanOrder。
  • 分销 B 链路固定为 ExternalDistributionPaidOrder -> Customer -> Order(paid) -> PlanOrder。
  • 低代码配置不能新增核心模型，不能让 PlanOrder 独立存在。
  • 低代码配置不能直接写 Customer、Order、PlanOrder。
  • 低代码配置不能让分销 paid order 进入 Clue。
  • 低代码配置不能把外部分销佣金、提现、退款资金流放入 SeedCRM 内部处理。
  • 所有接口仍必须执行 RBAC + ABAC，数据范围仍为 self / team / store / all / partner。

4. 接口定义

接口分为配置管理接口和运行时读取接口。配置管理接口用于草稿、校验、预览、发布；运行时接口只返回 UI 元数据，不执行业务写入。

接口用途权限
GET /lowcode/workspaces读取配置空间、环境和当前版本。lowcode:view
GET /lowcode/assets按配置类型、业务模块、风险等级查询配置资产。lowcode:view
POST /lowcode/assets/create-draft基于当前版本或模板创建草稿。lowcode:draft
POST /lowcode/assets/save-draft保存配置草稿，不影响线上。lowcode:draft
POST /lowcode/validate校验 schema、权限引用、字段引用、主链路、安全门禁。lowcode:draft
POST /lowcode/dry-run规则、Provider、Scheduler、待办、动作执行预览，不落核心表。lowcode:debug
POST /lowcode/preview-diff预览草稿与线上版本差异。lowcode:view
POST /lowcode/preview-permission按角色/用户预览菜单、按钮、字段、数据范围、脱敏。lowcode:view
POST /lowcode/preview-impact展示影响角色、门店、用户、模块和高风险项。lowcode:view
POST /lowcode/publish发布配置版本，写目标配置表、审计并失效缓存。lowcode:publish
POST /lowcode/rollback-preview生成回滚草稿和差异预览。lowcode:view
POST /lowcode/rollback以新版本方式回滚配置。lowcode:publish
GET /lowcode/versions查询配置版本。lowcode:view
GET /lowcode/audit-logs查询配置审计，敏感字段脱敏。lowcode:audit
GET /runtime-config/menu读取当前用户可见菜单。登录用户 + RBAC + ABAC
GET /runtime-config/page/{pageCode}读取页面布局、列表、字段、按钮元数据。原页面权限 + ABAC
GET /runtime-config/actions/{moduleCode}读取当前用户可用动作，不执行动作。原模块权限 + ABAC

5. 权限规则

RBAC

  • ADMIN：配置平台全量查看、草稿、校验、发布、回滚、审计。
  • INTEGRATION_ADMIN：Provider、Callback、Open API、Scheduler、接口调试相关配置草稿与 dry-run；生产发布需管理员复核。
  • INTEGRATION_OPERATOR：只读、dry-run、日志和健康检查，不可发布，不可切换 LIVE。
  • CLUE_MANAGER：可维护线索列表、自动分配、值班客服、工作台待办等低/中风险草稿，不发布全局权限。
  • STORE_MANAGER：可维护门店服务模板、门店绑定、门店人员视图类配置，不可扩大跨门店数据范围。
  • FINANCE：可读取薪酬与结算配置效果，不配置外部分销资金流。
  • PARTNER_APP：不可访问后台低代码平台，只能访问自身 partner 范围的 Open API。

ABAC

  • 运行时配置解析必须带 PermissionRequestContext。
  • 数据范围必须支持 self / team / store / all / partner，不得由前端参数提升。
  • 权限预览必须支持按角色、按用户、按门店、按 partner 查看最终结果。
  • 字段配置必须叠加敏感字段脱敏，不允许通过配置绕过。
  • 手机号、raw_data、token、secret、auth_code、签名、支付流水、外部订单原文必须脱敏。

高风险门禁

  • 开放导出、批量操作、重试、回放、LIVE 切换必须进入发布中心。
  • 字段脱敏放开、权限扩大、金额完整显示、raw_data 查看必须展示影响范围。
  • 生产发布必须填写发布原因并输入确认词。
  • 阻断项禁止发布，例如普通角色获得 raw_data 完整查看、动作绕过受控服务直接写核心表、管理员失去配置入口。

6. 流程

配置向导流程

设计器流程

发布流程

  1. 创建草稿：从模板或当前线上配置复制。
  2. 表单化配置：不直接编辑大 JSON，技术字段放入高级设置。
  3. 实时预览：看到菜单、列表、字段、按钮、表单变化。
  4. 系统校验：字段引用、权限引用、ABAC、脱敏、主链路、安全门禁。
  5. 权限预览：按角色/用户查看最终可见内容。
  6. 影响预览：展示影响角色、门店、用户、接口、缓存和风险项。
  7. 发布上线：写版本、目标配置表、审计，失效缓存。
  8. 回滚：从发布记录生成回滚草稿，预览后发布为新版本。

7. 可配置与不可配置清单

第一批可配置

  • 菜单、角色、权限点、数据范围绑定。
  • 字典、参数、状态中文名、颜色、排序。
  • 列表列、筛选项、默认排序、固定列、分页。
  • 表单字段、必填、只读、默认值、简单校验。
  • 按钮可见条件、权限点、确认文案、跳转入口。
  • 服务单模板、门店绑定、效果预览。
  • 工作台待办规则、角色入口、风险提醒。
  • Provider Mapping、Callback、Scheduler 频率、重试、熔断、MOCK/LIVE 门禁。

明确不可配置

  • 新增核心业务模型或核心业务表。
  • 任意 SQL、任意脚本、任意接口编排。
  • 直接写 Customer、Order、PlanOrder。
  • 修改订单/排档核心状态机。
  • PlanOrder 脱离 Order 独立创建。
  • 分销 paid order 进入 Clue。
  • Scheduler 绕过入站服务直写业务表。
  • 外部分销佣金、提现、退款资金流进入 SeedCRM。

8. 落地顺序

阶段目标复用现有能力验收标准风险
L0低代码边界文档与配置类型白名单。AGENT.md、CONTEXT.md。可配置/不可配置边界清晰，主链路红线明确。中
L1配置版本底座：草稿、diff、validate、publish、rollback、audit。SystemFlow、SystemConfig。所有配置变更可预览、可回滚、可审计。高
L2菜单权限配置化升级。SystemAccess、Permission。新增菜单父级可选择，权限发布前可按角色/用户预览。高
L3字典、字段、列表、页面运行时配置。SystemSettingView、字典中心、当前表格页面。常见字段显示、筛选、中文化不再改代码。中
L4表单模板与按钮动作配置。ServiceFormTemplate、PlanOrder。服务单设计支持模板、预览、发布；按钮不能直写核心表。高
L5工作台待办规则配置。Workbench。待办只做提醒和跳转，不改变业务状态。中
L6Provider / Scheduler 受控低代码。Scheduler、Provider、Outbox、异常队列、接口调试。Mapping、频率、重试、LIVE 切换必须 dry-run + 发布。高

9. 测试与验收

  • 草稿不影响线上运行。
  • 发布成功后，目标配置、版本、审计、缓存失效一致。
  • 发布失败整体回滚。
  • 回滚以新版本发布，不覆盖历史。
  • 低代码运行时接口只返回元数据，不做业务写入。
  • 按钮动作只调用已有受控业务接口。
  • 权限预览结果与真实登录菜单、按钮、字段、数据范围一致。
  • 字段展示配置不能突破手机号、raw_data、token、secret、auth_code、支付流水等脱敏策略。
  • 分销 paid order 在任何配置下都不能进入 Clue。
  • Scheduler 在任何配置下都不能直接写 Customer、Order、PlanOrder。
  • dry-run 不落核心业务表，不调用真实外部 LIVE。
  • Provider Mapping 改动必须能预览字段映射、状态映射、幂等结果。
  • PlanOrder 不能通过表单模板或按钮配置独立创建。
  • 生产发布、高风险权限扩大、导出、回放、LIVE 切换必须二次确认并审计。

10. 禁止行为

  • 禁止把受控低代码升级成任意建表、任意 SQL、任意脚本平台。
  • 禁止配置改变 Clue -> Customer -> Order -> PlanOrder 主链路。
  • 禁止配置让分销 paid order 进入 Clue。
  • 禁止配置直接创建或修改 Customer、Order、PlanOrder。
  • 禁止配置让 PlanOrder 不绑定 Order。
  • 禁止配置绕过 RBAC + ABAC。
  • 禁止配置绕过统一入站服务、Outbox、异常队列和 Scheduler 安全门禁。
  • 禁止导入配置覆盖生产密钥或切换 LIVE 模式。
  • 禁止审计日志保存明文 secret、token、auth_code、raw_data。
  • 禁止把外部分销佣金、提现、退款资金流配置进 SeedCRM。

11. 需要你确认的问题

建议先确认以下 3 点，再进入代码落地：1）是否接受“受控低代码”命名与边界；2）P0 是否先做模板化配置而不是拖拽式设计器；3）是否优先从菜单权限和页面列表配置开始，而不是先做 Provider/Scheduler。
