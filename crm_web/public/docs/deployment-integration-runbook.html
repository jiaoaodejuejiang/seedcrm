SeedCRM 上线与联调运行手册

本文用于方案 B 上线前检查、服务器部署、外部分销联调和问题回退。方案 B 主链路固定为：外部分销已支付订单 → Customer → Order(已支付) → PlanOrder → Order used → Outbox 回推。

不进入 Clue 受控 Open API Dry-run 不落库 Outbox 异步回推 Swagger 受保护

1. 上线前硬性约束

  • 外部分销已支付订单只能通过 /open/distribution/events 入站，不允许进入 Clue。
  • API Controller 和 Scheduler 都必须复用 DistributionEventIngestService，不得直接写 Customer / Order / PlanOrder。
  • 联调工作台必须使用 dry-run，不能写核心业务表。
  • 履约完成只能先写 scheduler_outbox_event，再由 Scheduler 异步回推外部分销系统。
  • 生产环境必须关闭本地 Swagger 免 token：SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN=false。

2. 服务器环境变量

变量 用途 生产建议
SPRING_PROFILES_ACTIVE 后端运行环境 生产建议设置为 prod，默认关闭 Swagger 并关闭本地免 token
SEEDCRM_DATASOURCE_URL 后端数据库连接 使用正式 MySQL 地址，禁止连接本地测试库
SEEDCRM_DATASOURCE_USERNAME 数据库账号 最小权限账号，至少区分生产和测试
SEEDCRM_DATASOURCE_PASSWORD 数据库密码 通过服务器密钥管理或环境变量注入，不写入仓库
SEEDCRM_SYSTEM_BASE_URL 后台系统访问域名 例如 https://crm.example.com
SEEDCRM_API_BASE_URL API、回调、Swagger 示例地址域名 例如 https://api.example.com，必须外网可访问
SEEDCRM_OPENAPI_ENABLED 是否启用 Swagger UI / OpenAPI JSON 需要联调时可启用；生产公网建议配合网关鉴权
SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN 本地访问 Swagger 是否免 token 生产必须设置为 false 
SPRING_PROFILES_ACTIVE=prod
SEEDCRM_SYSTEM_BASE_URL=https://crm.example.com
SEEDCRM_API_BASE_URL=https://api.example.com
SEEDCRM_OPENAPI_ENABLED=false
SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN=false

3. 域名与回调配置

  1. 进入【系统设置 → 基础配置 → 域名配置】，保存系统基础域名和 API 域名。
  2. 进入【系统设置 → 调度中心 → 分销接口】，确认“生产联调信息”中的入站、回查、对账和 Swagger 地址已经变为线上域名。
  3. 在外部分销系统中配置 SeedCRM 入站地址：{API 域名}/open/distribution/events。
  4. 在 SeedCRM 分销接口配置中填写外部分销系统的履约回推目标、状态回查路径、对账拉取路径和 AppSecret。
  5. 若涉及抖音、企业微信、支付回调，各配置页只能编辑路径或三方地址，基础域名必须来自域名配置。
如果页面仍显示 127.0.0.1，不要进入真实联调，先检查环境变量和域名配置保存结果。

4. Swagger 与联调工作台

Swagger UI

用于查看接口定义、字段结构和 OpenAPI 分组。远程访问只允许管理员、集成管理员、集成操作员。

入口：{API 域名}/swagger-ui.html

联调工作台

用于 dry-run 验证签名、字段映射、状态映射、Customer 匹配和 Order 幂等结果。

要求：不能写入 Customer / Order / PlanOrder。

生产环境不要把 Swagger/OpenAPI 直接暴露给公网未授权访问。若通过 Nginx/Gateway 暴露，必须叠加鉴权、IP 白名单或 VPN。

5. 方案 B 联调步骤

  1. 在 MOCK 模式下保存分销配置，打开联调工作台，执行 DISTRIBUTION_ORDER_PAID dry-run。
  2. 确认结果显示：不写 Clue、预览 Customer 匹配、预览 Order 幂等、PlanOrder 不直接创建。
  3. 切换 LIVE 模式前补齐 API 域名、AppId、AppSecret、入站路径、回推目标、状态回查路径、对账路径。
  4. 外部系统发起真实 distribution.order.paid 入站，确认返回 traceId、customerId、orderId。
  5. 进入【会员信息】确认分销成交会员显示为 Customer 视角，不出现客资/线索/公海话术。
  6. 进入【顾客排档】预约门店档期，再进入【门店服务】完成核销、服务确认和订单完成。
  7. 确认 crm.order.used 进入 Outbox，Scheduler 成功回推外部分销系统。
  8. 若回推失败，进入异常队列或 Outbox 重试，不回滚本地订单履约。

6. 上线验收清单

  • 域名配置保存后，分销接口页显示的所有地址都是线上域名。
  • Swagger UI / OpenAPI JSON 远程访问需要后台 token，普通业务角色和 partner_app 不能访问。
  • 联调工作台 dry-run 不写核心业务表。
  • 重复 X-Idempotency-Key 或重复外部订单不会重复创建 Order。
  • 历史重复 callback log 已治理，或形成上线风险确认。
  • PlanOrder 完成后生成 Outbox，而不是同步阻塞等待外部系统。
  • 外部分销退款、取消只记录状态影响，不在 SeedCRM 处理资金流、分佣、提现。
  • 数据库、后端、前端构建和浏览器冒烟验证均通过。

7. 常见问题处理

现象 处理建议
外部系统回调 403 检查 X-Partner-Code、签名头、时间戳窗口、nonce 是否重复，确认分销配置已启用。
Swagger 线上无法打开 检查 SEEDCRM_OPENAPI_ENABLED、后台 token、网关鉴权和 SEEDCRM_OPENAPI_ALLOW_LOCAL_WITHOUT_TOKEN=false。
入站进入异常队列 查看冲突字段、外部订单号、本地订单号、TraceId 和处理建议；修复配置或数据后再重试。
Outbox 回推失败 检查外部分销回调地址、AppSecret、网络连通性和响应码；修复后执行单条重试或队列处理。
