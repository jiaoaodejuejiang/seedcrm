SeedCRM 宝塔面板 Docker 部署说明

面向使用宝塔面板管理服务器的部署方式。推荐使用宝塔的 Docker / Docker Compose 能力部署，本系统默认前端端口为 8003，后端接口端口为 8004。

前端 8003 后端 8004 MySQL 8.x Compose 编排

1. 宝塔面板准备

  1. 在宝塔面板中安装 Docker 管理器，确认服务器可以执行 docker compose version。
  2. 放行安全组和宝塔防火墙端口：8003、8004。MySQL 默认只绑定服务器本机 127.0.0.1:3306，不建议对公网放行。
  3. 建议先创建站点域名，例如 crm.example.com 指向前端，api.example.com 指向后端。
  4. 如果没有域名，测试阶段可先用 http://服务器IP:8003 和 http://服务器IP:8004 联调。

2. 上传代码与配置

在宝塔文件管理中创建目录，例如 /www/wwwroot/seedcrm，上传整个项目代码。

cd /www/wwwroot/seedcrm
cp .env.example .env

编辑 .env，至少修改数据库密码和域名：

SEEDCRM_MYSQL_PASSWORD=请替换为强密码
SEEDCRM_MYSQL_ROOT_PASSWORD=请替换为强密码
SEEDCRM_SYSTEM_BASE_URL=http://服务器IP:8003
SEEDCRM_API_BASE_URL=http://服务器IP:8004
SEEDCRM_FRONTEND_PORT=8003
SEEDCRM_BACKEND_PORT=8004
SEEDCRM_ALLOW_CLEAR_TEST_DATA=false

SEEDCRM_ALLOW_CLEAR_TEST_DATA 默认必须保持 false。只有在确认连接的是测试库、且已备份后，才可临时改为 true 执行真实测试数据清理。

3. 启动容器

可在宝塔终端执行，也可在 Docker Compose 项目中选择本项目的 docker-compose.yml。

docker compose build
docker compose up -d
docker compose ps
docker compose logs -f backend
服务容器端口说明
frontendseedcrm-frontend8003 -> 80后台前端页面，内部 Nginx 会把 /api 代理到后端。
backendseedcrm-backend8004 -> 8004Spring Boot API 服务。
mysqlseedcrm-mysql127.0.0.1:3306 -> 3306仅服务器本机可访问，正式环境不要对公网开放。

4. 宝塔反向代理建议

前端站点

宝塔站点 crm.example.com 反代到：

http://127.0.0.1:8003

后端 API 站点

宝塔站点 api.example.com 反代到：

http://127.0.0.1:8004

开启 HTTPS 后，把【系统设置 / 基础配置 / 域名配置】更新为：

系统基础域名：https://crm.example.com
API 域名：https://api.example.com

5. 上线工具检查

  1. 访问 http://服务器IP:8003，使用管理员账号登录。
  2. 进入【系统设置 / 基础配置 / 上线工具】，先运行上线预检。
  3. 清理测试数据只能在 TEST/STAGING 环境执行，先 dry-run，确认词为 CLEAR_TEST_DATA；真实删除还需要 SEEDCRM_ALLOW_CLEAR_TEST_DATA=true。
  4. 正式环境初始化确认词为 INIT_PROD_SYSTEM，并且不会自动把接口切回 MOCK。

6. Scheme B 接口联调

  • 外部分销已支付订单入站地址：{API 域名}/open/distribution/events。
  • 分销订单不进入 Clue，只通过受控入站服务创建或匹配 Customer，并创建 Order(已支付)。
  • 状态回查、对账拉取先执行“预检（不入库）”，再执行真实任务。
  • PlanOrder finished 后写 Outbox，调度异步回推 crm.order.used。

7. 不建议上线的情况

  • 宝塔防火墙或云安全组未放行 8003/8004，外部平台无法回调。
  • 线上域名仍配置为 127.0.0.1、localhost 或测试 IP。
  • Swagger/OpenAPI 在公网开启但没有鉴权、IP 白名单或 VPN 限制。
  • 抖音 auth_code 已过期，但仍继续保存或测试真实拉取。
  • 没有先执行上线工具 dry-run，或没有确认当前连接测试库，就直接打开真实清理开关。
